Logo

Our logo is at the heart of PEACH and should be used following the strict guidelines that follow.

Which Logo To Use?

This is up to you to decide - However be sure to match the logo to the theme of your application. For example, if you use the purple logo this must match the navbar and other components on the page.

You are able to colour components in different shades of the chosen logo.


The Clearspace Rule

Typography

Primarily all applications should be built using Karla font for text.
Text is read by all users and must be consistent across platforms.

To use Karla font, simple copy and paste the link below into your html head tag.

<link href='https://fonts.googleapis.com/css?family=Karla' rel='stylesheet' type='text/css'>

Add the font name to your CSS styles just as you'd do normally with any other font.

*className*{ font-family: Karla; }

Buttons

Use buttons throughout applications using the btn-peach-*colour*-*shade*class.

Do's and Don'ts

  • Do use disabled attribute if button cannot be clicked.
  • Do use same colour button as chosen navbar.
  • Do not alter button sizes.

Filled Example Usage









<button class="btn-peach-blue-med"> Submit </button>
<button class="btn-peach-green-dark"> Submit </button>

Outlined Example Usage

<button class="btn-peach-orange-dark-outline"> Submit </button>
<button class="btn-peach-sand-dark-outline"> Submit </button>

Dropdowns

This template consists of two parts: the dropdown toggle and the dropdown menu, using the dropdown-toggleclass and dropdown-menuclass respectively.

Do's and Don'ts

  • Do hide the dropdown menu when the toggle is off.

Navbar

Use the same navbar throughout applications using the nav-peach-*colour* class.

Do's and Don'ts

  • Do make sure to use the collapsable navbar for mobile.
  • Do not change the height of navbar
  • Do not change the brand logo padding. Must adhere to clear space rule.
